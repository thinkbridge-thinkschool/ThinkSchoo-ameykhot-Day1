Prevention Table — Which Isolation Level Fixes Each Anomaly

Anomaly What Happens Lowest Level That Prevents It How It Works
Dirty Read Session 2 reads data Session 1 changed but NOT yet committed READ COMMITTED Blocks reads on rows with open write locks — waits until writer commits or rolls back
Non-Repeatable Read Session 2 reads same row twice, gets different values REPEATABLE READ Holds shared locks on read rows until transaction ends — other sessions cannot UPDATE those rows
Phantom Read Session 2 runs same range query twice, gets different row count SERIALIZABLE Holds key-range locks on the search predicate — other sessions cannot INSERT rows that match the range
