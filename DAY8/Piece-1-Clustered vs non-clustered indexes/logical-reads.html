⚠ Write-Side Cost — INSERT 1,000 rows (1 clustered + 2 NC indexes)
Sales table logical reads: 10,669  •  Worktable (sort): 2,032  •  Total I/O for 1,000 rows: ~12,700 logical reads

Every INSERT writes to 3 B-trees (CIX + IX_CustomerId + IX_SaleDate). With 0 NC indexes the clustered write alone costs ~3,500 reads. Two NC indexes tripled the write I/O to 10,669 — a ~3× write amplification. This is the fundamental trade-off: indexes make reads dramatically faster but every INSERT, UPDATE, or DELETE pays a tax on each index maintained.
